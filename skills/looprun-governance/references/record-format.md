# Record format

A proof record is one file at `governance/proofs/YYYY-MM-DD-<slug>.md`, generated by
`pnpm proofs:record` (never hand-written) and indexed into `governance/MATRIX.md`. The frontmatter is a
**flat `key: value` contract** â€” parsed by a trivial line reader between two `---` fences, no YAML
library, no nesting â€” so it never merge-conflicts and is trivial to gate on.

## The contract

```
---
date: 2026-07-15
slug: example-slug
change_kind: guard | runtime | skill | docs
target: <kind or component>
summary: One line.
isolated: 125/125
collective: 34/34
coverage: 27/27
certified_models: n/a
slm_canary: n/a
verdict: PASS
suite_cmd: pnpm proofs:run
---
```

| key | meaning |
|---|---|
| `date` | `YYYY-MM-DD`; also the filename prefix |
| `slug` | kebab-case identifier; also the filename tail |
| `change_kind` | `guard` Â· `runtime` Â· `skill` Â· `docs` (the head of `--scope`) |
| `target` | the guard kind or component touched (`â€”` when not applicable) |
| `summary` | the one-line change (the matrix "Change" column) |
| `isolated` | `pass/total` across L1 + L3 lanes |
| `collective` | `pass/total` across the collective non-interference lane |
| `coverage` | `covered/kinds` â€” guard kinds whose completeness describe fully passed |
| `certified_models` | the per-model certification data for the artifact this change ships: `;`-joined `model:scoreĂ—reps` entries (e.g. `flash-lite:61/61Ă—3; local-35b:57/61Ă—1+band`), or `n/a` when the change ships no certified bundle (most runtime/guard/skill changes). Always emitted by `--certified`; defaults to `n/a`. Legacy records without the key render `n/a` in the matrix |
| `slm_canary` | `n/a` or `n/n` from the report-only SLM lane (advisory, never a gate) |
| `verdict` | `PASS` iff every proof passed; otherwise `FAIL` |
| `suite_cmd` | the command that produced the tallies (`pnpm proofs:run`) |

The gate accepts a governed change only when the diff adds/modifies a record whose frontmatter parses
and reads `verdict: PASS`.

Generate a real one with:

```bash
pnpm proofs:run
pnpm proofs:record -- --slug arg-format-handle-shape \
  --change "argFormat rejects malformed @handles before schedulePost runs" \
  --scope guard:argFormat --slm "3/3" --certified "flash-lite:61/61Ă—3; local-35b:57/61Ă—1+band"
```

`--certified` is optional (defaults to `n/a`); pass it only for a change that ships or re-certifies a
measured deployment bundle. Skill/docs/runtime changes that touch no certified artifact leave it `n/a`.

The documented `pnpm proofs:record -- â€¦` idiom is safe: a bare `--` passthrough separator is ignored by
the generator, so args parse the same whether or not the package manager forwards it.

**Proof cases section.** For a `skill`/`docs` scope (or any change flagged `--no-proof-cases`) the
generator emits `Proof cases: n/a (docs/skill-only change; guard runtime unchanged; `pnpm proofs:run`
N/N unchanged).` instead of the guard proof-case authoring boilerplate â€” a docs-only change touches no
guard/runtime source, so authoring positive/negative/neutral guard cases does not apply. `guard`/`runtime`
scopes keep the authoring boilerplate.
