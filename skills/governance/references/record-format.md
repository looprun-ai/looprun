# Record format

A proof record is one file at `governance/proofs/YYYY-MM-DD-<slug>.md`, generated by
`pnpm proofs:record` (never hand-written) and indexed into `governance/MATRIX.md`. The frontmatter is a
**flat `key: value` contract** — parsed by a trivial line reader between two `---` fences, no YAML
library, no nesting — so it never merge-conflicts and is trivial to gate on.

## The contract

```
---
date: 2026-07-15
slug: example-slug
change_kind: guard | runtime | skill | docs
target: <kind or component>
summary: One line.
isolated: 125/125
collective: 34/34
coverage: 27/27
slm_canary: n/a
verdict: PASS
suite_cmd: pnpm proofs:run
---
```

| key | meaning |
|---|---|
| `date` | `YYYY-MM-DD`; also the filename prefix |
| `slug` | kebab-case identifier; also the filename tail |
| `change_kind` | `guard` · `runtime` · `skill` · `docs` (the head of `--scope`) |
| `target` | the guard kind or component touched (`—` when not applicable) |
| `summary` | the one-line change (the matrix "Change" column) |
| `isolated` | `pass/total` across L1 + L3 lanes |
| `collective` | `pass/total` across the collective non-interference lane |
| `coverage` | `covered/kinds` — guard kinds whose completeness describe fully passed |
| `slm_canary` | `n/a` or `n/n` from the report-only SLM lane (advisory, never a gate) |
| `verdict` | `PASS` iff every proof passed; otherwise `FAIL` |
| `suite_cmd` | the command that produced the tallies (`pnpm proofs:run`) |

The gate accepts a governed change only when the diff adds/modifies a record whose frontmatter parses
and reads `verdict: PASS`.

## Filled example

```
---
date: 2026-07-15
slug: arg-format-handle-shape
change_kind: guard
target: argFormat
summary: argFormat rejects malformed @handles before schedulePost runs
isolated: 128/128
collective: 35/35
coverage: 27/27
slm_canary: 3/3
verdict: PASS
suite_cmd: pnpm proofs:run
---

# Proof record — argFormat rejects malformed @handles before schedulePost runs

**Scope:** `guard:argFormat` · **Date:** 2026-07-15 · **Verdict:** PASS

## What changed
Tightened the argFormat instance on schedulePost so a handle missing its leading '@' is vetoed with a
correction, while an absent/empty handle stays deferred to argRequired.

## Proof cases
- positive: args.handle = 'acme' → veto `input:argFormat:schedulePost`
- negative: args.handle = '@acme' → null
- neutral: args.handle absent → null (deferred to argRequired, declared in alsoFires)
- l3: scripted turn attempts schedulePost with 'acme' → veto tag, model recovers with '@acme'

## Results
| lane | pass/total |
|---|---|
| isolated (L1 + L3) | 128/128 |
| collective | 35/35 |
| coverage (kinds fully proven) | 27/27 |
| **all** | **190/190** |

## SLM canary (advisory)
Report-only small-local-model run: 3/3.

## Verdict & residuals
**PASS.**
No residuals.
```

Generate a real one with:

```bash
pnpm proofs:run
pnpm proofs:record -- --slug arg-format-handle-shape \
  --change "argFormat rejects malformed @handles before schedulePost runs" \
  --scope guard:argFormat --slm "3/3"
```
