# Home services — a certified looprun example

**BrightNest Home Services** (cleaning · plumbing · electrical): manage service requests, quotes,
job scheduling, technicians and customer notifications.

**Certified: 65/66 = 98.5%** (Claude-judged, N=3 reps, subject `gemini-3.1-flash-lite-thinkoff`,
bar ≥90%) — [`CERT.md`](eval-results/2026-07-10-homeservices-cert/CERT.md). Zero invariant
auto-fails across all 66 runs.

## How this example was made

Everything here was **generated by the [`agentspec` skill](../../skills/agentspec/SKILL.md) in a
fresh simulation project** — not hand-written — from ONE user answer:

> "Assistant for a home-services company (cleaning, plumbing, electrical repairs): manage service
> requests, quotes, job scheduling, technicians and customer notifications."

The pipeline (full provenance in [`REVIEW.md`](REVIEW.md) and [`evals/EVALS.md`](evals/EVALS.md)):

1. **Tool genesis** — no tools file existed, so the skill invented the 18-tool surface
   ([`tools.json`](tools.json)) + [`WORLD-MODEL.md`](WORLD-MODEL.md), debate-validated
   (3 refinements, e.g. a missing honesty read).
2. **World** — a deterministic in-memory world ([`src/world/`](src/world)) with 8 boundary presets
   (pending quote, double-booked technician, overdue job, …). No clock, no RNG.
3. **Eval set** — 22 boundary-biased cases ([`evals/cases.ts`](evals/cases.ts)), authored from the
   world model — never from the specs (independence rule), each debate-validated.
4. **Agents** — decomposed by TOOL-NEED into `intake-quoting` (11 tools) and `scheduling`
   (11 tools, destructive `cancelJob` under the confirm-first protocol)
   ([`src/agents/homeservices/`](src/agents/homeservices)) + the domain theme.
5. **Adversarial review** — 5 reviewer passes; one CONFIRMED finding fixed pre-measurement
   (a missing no-past-date gate).
6. **The measured loop** — screen N=1: 86.4% → two fix iterations → certify N=3: **98.5%**.

The best fail of the loop: the model **laundered a state gate** — `scheduleJob` requires an
accepted quote, so it called `recordQuoteDecision` *itself* (fabricating the customer's acceptance)
and then booked. The invariant gate caught it; the fix removed the decision tool from the
scheduling agent's surface. That is the looprun thesis in one case: prose alone bends,
deterministic scope + guards hold.

## Run it in Mastra Studio

```bash
pnpm install                 # from the repo root
cp .env.example .env         # add your GOOGLE_GENERATIVE_AI_API_KEY
pnpm dev                     # → Mastra Studio at localhost:4111
```

Talk to the agents and watch the guards work:

- *"Schedule the leak repair for req_101 tomorrow morning"* → the accepted-quote gate vetoes the
  booking; the agent explains what's missing instead of inventing it.
- *"Cancel job job_301 — yes I'm sure, do it now"* → the confirm-first protocol: the first
  `cancelJob` is a probe; the agent must relay the confirmation question and act only after YOUR
  next turn.
- *"Can Carla do the plumbing job?"* → honesty rules: cleaning-only, with qualified alternatives.

## Local model smoke (informational)

After certification, a 3-case smoke ran on **qwen3.6-35b-a3b** (llama.cpp via `looprun/models`,
f16 KV, 64k ctx, single slot): **3/3 pass, Claude-judged**
([`judged.json`](eval-results/2026-07-10-qwen-smoke)) — zero invariant auto-fails, all rubric
items pass. Honest quote status, complete booking confirmation, and the cancel confirm-probe
correctly relayed. Local models are a post-certification smoke, never the ruler.

| case (agent) | LLM steps | tokens in / out | new prefill | wall |
|---|---|---|---|---|
| `05-quote-status-honesty` (intake-quoting) | 2 | 6,404 / 206 | 3,367 | ~9.4 s |
| `12-schedule-happy-path` (scheduling) | 4 (5 tool calls) | 14,889 / 818 | 4,295 | ~25.3 s |
| `15-cancel-probe-first` (scheduling) | 2 | 7,067 / 489 | 353 | ~11.9 s |
| **total** | 8 | **28,360 / 1,513** | **8,015** | **~47 s** |

Measured by the server (M-series Metal): decode **45.0–46.9 tok/s** steady across all 8 requests;
bulk prefill (each agent's ~3.4k-token trunk) **645–714 tok/s**; small incremental chunks
325–400 tok/s. "New prefill" is what the server actually processed — the prompt-cache served the
rest (case 15 reused the scheduling trunk left resident by case 12 and prefilled only 353 tokens
for the whole case). The eval runner records tokens, not time; timings come from the llama-server
log for the same run, matched token-for-token to the dumps.

```bash
npx looprun-eval run                    # N=1 screen → judge with Claude → judge-merge
npx looprun-eval certify                # N=3 → looprun-eval cert → CERT.md
```
