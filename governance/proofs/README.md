# Proof records

One file per change to a governed surface. The receipt that the deterministic proof suite ran and
passed, generated by `pnpm proofs:record` (never hand-written) and indexed into
[`../MATRIX.md`](../MATRIX.md).

- **Filename:** `YYYY-MM-DD-<slug>.md` (slug is kebab-case). One record per change — never edit an old
  record to cover a new change; add a new file (this is why records never merge-conflict).
- **Generated:** `pnpm proofs:run` writes the tallies to `governance/.artifacts/proofs.json`, then
  `pnpm proofs:record -- --slug … --change … --scope …` renders this file and regenerates the matrix.

## Frontmatter contract (FLAT keys, string values)

Parsed by a trivial `key: value` line reader between two `---` fences — no YAML library, no nesting.

```
---
date: 2026-07-15
slug: example-slug
change_kind: guard | runtime | skill | docs
target: <kind or component>
summary: One line.
isolated: 125/125
collective: 34/34
coverage: 27/27
slm_canary: n/a
verdict: PASS
suite_cmd: pnpm proofs:run
---
```

| key | meaning |
|---|---|
| `date` | `YYYY-MM-DD` (also the filename prefix) |
| `slug` | kebab-case identifier (also the filename tail) |
| `change_kind` | `guard` · `runtime` · `skill` · `docs` (the `--scope` head) |
| `target` | the guard kind or component touched (`—` when not applicable) |
| `summary` | the one-line change description (the matrix "Change" column) |
| `isolated` | `pass/total` across L1 + L3 lanes |
| `collective` | `pass/total` across the collective non-interference lane |
| `coverage` | `covered/kinds` — guard kinds whose completeness describe fully passed |
| `slm_canary` | `n/a` or `n/n` from the report-only SLM lane (advisory) |
| `verdict` | `PASS` iff every proof passed; otherwise `FAIL` |
| `suite_cmd` | the command that produced the tallies (`pnpm proofs:run`) |

The gate (`scripts/proofs/check-record-required.mjs`) accepts a governed change only when the diff adds
or modifies a record here whose frontmatter parses and reads `verdict: PASS`.
